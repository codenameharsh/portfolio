Assembly: End-Game

Case Study | A word-guessing game with a mysterious twist

Duration: 1 week

Project Overview

Assembly: End Game is an interactive, single-player word-guessing game crafted with mystery and tension at its core. Built using React.js, JavaScript (ES6+), and CSS3, it challenges users to decode a hidden word within eight attempts—before an enigmatic “Assembly” takes over. The interface is sleek, the interactions responsive, and the experience intentionally atmospheric.

This game was a development exercise and a design problem disguised as play. How do you make something familiar (a word game) feel new, immersive, and emotionally charged?

Project Goals

From the start, I wanted to go beyond just replicating mechanics like Wordle or Hangman. My goals were:

  • Design an emotionally engaging gameplay loop—something that felt suspenseful and even cinematic.
  • Craft a visually dynamic interface that communicates feedback clearly and meaningfully.
  • Apply my growing React and JS skills in a context where design and logic coexisted seamlessly.
  • Ensure accessibility and responsiveness across all devices, from desktops to mobile phones.

Research & Learning Process

This project began as a learning challenge on Scrimba, where I was diving deep into React.js and component-based design. But rather than simply follow tutorials, I chose to personalize it. I started by analyzing:

  • User interaction patterns in casual games: What keeps users engaged? What frustrates them?
  • Tension-building mechanics in games and media: How do minimal visuals and sounds affect perception?
  • Accessibility in interactive applications: How can I ensure clarity for different players, including those on small screens?

I also gathered inspiration from:

  • The mood and ambiance of sci-fi thrillers and escape-room challenges.
  • The emotional pacing of storytelling games, where small UI shifts build dramatic arcs.

Design Process

    Problem Definition
  • How might I design a simple game mechanic in a way that feels new, narratively rich, and emotionally reactive, while also using it as a learning sandbox for React.js?
    Ideation

    The primary goal was to design a fun yet challenging word-guessing game with engaging user interactions. Key considerations included:

  • Interactive Gameplay: Encouraging user engagement through an intuitive interface.
  • Responsive UI: Ensuring a seamless experience across desktops, tablets, and mobile devices.
  • Visual & Audio Feedback: Creating a dynamic UI that reacts to user inputs with animations and effects.
    Wireframes & Prototypes
  • I sketched basic layouts to explore how a user would visually process feedback.
  • Prototyped game board placements: centered for focus, with guessed letters and attempts clearly visible.
  • Iterated on color choices to evoke suspense (dark tones, contrasting feedback).
  • Designed responsive variations using media queries and flex/grid for optimal mobile experience.
    Coding Challenges
  • Managing state logic for guesses, attempts, and feedback was a steep learning curve.
  • Balancing performance with dynamic updates was key to smooth gameplay.
  • Building a confetti animation for wins and a multi-language farewell effect for losses stretched both my creative and coding muscles.
    Tech Stack
  • React.js: State management, reusable UI components.
  • JavaScript (ES6+): Game logic, conditionals, string manipulations.
  • CSS3: Animations, responsiveness, and creating the "mood" visually.

Key Features

  1. Guessing Mechanics: Reveal correct letters; penalize wrong guesses with animated feedback.
  2. Threat Progression: The mysterious “Assembly” visually escalates with each wrong guess—adding narrative tension.
  3. Confetti Win Animation: A celebratory payoff that rewards users for decoding the word.
  4. Multi-language Farewell System: As failure nears, the UI issues farewells in different languages—creating emotional distance and finality.
  5. Responsive Design: Carefully tuned for a smooth experience on desktops, tablets, and phones.
  6. Color-coded Feedback: Visual cues for correctness help build intuition and clarity.

Final Outcome & Impact

Assembly: End Game became much more than just a side project—it became a canvas to explore how emotion, interface, and interaction can come together in a playful experience. The final product:

  • Delivered a fun, immersive, and suspense-driven guessing game
  • Showcased my ability to merge design intent with code execution using React.
  • Demonstrated my focus on responsive and accessible UI design.
  • Highlighted how small design decisions (like farewell messages or confetti) can transform a simple game into a memorable micro-experience.

Final Thoughts

This project taught me that design is storytelling, even in a small game. Every decision—from typography to button placement to how feedback is presented—carries weight. More than that, it solidified my belief in designing with emotion. A good interface isn't just functional; it resonates.

    Key Takeaways:
  • Balancing functionality and mood requires intent.
  • Emotionally reactive design elevates even the simplest mechanics.
  • Designing around a fictional narrative gave more depth to UI decisions.
    Future Improvements:
  • Add custom word lists or themes to extend replayability.
  • Introduce accessibility features like screen reader support and high-contrast modes.
  • Enhance sound design to further build tension.
